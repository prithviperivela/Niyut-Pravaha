1️⃣ Core Goal of the System

Your project aims to build a Flow-Aware Music Recommendation Engine.

Unlike typical recommendation systems that optimize similarity, this system optimizes:

smooth song-to-song transition while maintaining pattern continuity.

The system works like a DJ-style sequencer.

Instead of:

song → similar song

It tries to produce:

song → next song with minimal perceptual disturbance

So the key problem being solved is:

Minimize transition disturbance between songs while maintaining pattern continuity.

2️⃣ Three-Layer Architecture

Your system will operate in three decision layers.

Layer 1 — Pattern Similarity (Macro Filtering)
Goal

Filter songs that match the overall acoustic pattern of the selected song.

This drastically reduces the search space.

Data Used

Spotify audio_features

Key features:

Feature	Meaning
tempo	BPM of song
energy	intensity
loudness	overall volume
danceability	rhythm stability
acousticness	calmness
instrumentalness	instrumental presence
speechiness	spoken content
valence	emotional positivity
key	musical key
mode	major/minor
time_signature	rhythm structure
Processing

1️⃣ Normalize features
2️⃣ Create feature vector

Example:

feature_vector =
[energy, tempo, loudness, danceability, acousticness, valence]
Similarity Metric

We use:

Cosine Similarity

similarity(A,B)

This gives a candidate pool of similar songs.

Example:

2500 songs → 100 candidate songs
Layer 2 — Energy Envelope Pattern Matching
Goal

Match dynamic energy evolution of songs.

Two songs may have the same average energy but very different structure.

Example patterns:

Song A
low → build → drop → sustain

Song B
high → sustain → drop

Even if average energy is same, patterns differ.

Data Used

Spotify audio_analysis

We extract:

section loudness

segment loudness

This lets us approximate the energy curve over time.

Processing

1️⃣ Extract loudness per segment
2️⃣ Normalize time length
3️⃣ Convert to fixed length vector

Example:

envelope_vector = [e1, e2, e3 ... e100]

Each element = energy at that time position.

Similarity Metric

Compare envelopes using:

Euclidean Distance

shape_distance(A,B)

Lower distance → similar wave pattern.

This reduces candidates further.

Example:

100 songs → 20 songs
Layer 3 — Transition Continuity Optimization
Goal

Minimize transition shock between songs.

We align:

End energy of song A
Start energy of song B
Calculation

Take small windows:

end_A = last k envelope frames
start_B = first k envelope frames

Compute:

transition_gap =
|mean(end_A) - mean(start_B)|

Lower gap = smoother transition.

Final Ranking Function

Final next-song score:

score =
α * macro_similarity
- β * shape_distance
- γ * transition_gap

Weights tuned experimentally.
