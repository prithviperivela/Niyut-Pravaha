The system can be understood as three main layers plus a data pipeline.

1️⃣ Data Layer (Foundation)

This is where Phase 1 happens.

Data Source

Spotify Web API.

Endpoints we will use:

Endpoint	Purpose
playlist_tracks	collect songs
audio_features	acoustic attributes
audio_analysis	structural song data

Later we add:

2️⃣ Layer 1 – Pattern Similarity
Goal

Find songs with similar acoustic patterns.

We convert the features into a vector representation.

Example:

feature_vector =
[energy, tempo, loudness, danceability,
 acousticness, instrumentalness,
 speechiness, valence]
Method

We compute:

cosine_similarity(songA, songB)

This filters the dataset:

2500 songs → ~100 candidates
3️⃣ Layer 2 – Energy Envelope Matching

This compares how the energy evolves through the song.

Example energy shapes:

Song A
low → build → drop → sustain

Song B
low → build → drop → sustain

These feel naturally compatible.

Data Used

Spotify audio_analysis

From this we extract:

segment loudness
section loudness

We convert this into:

envelope_vector = [e1, e2, e3 ... e100]

Now we compare shapes.

Method
shape_distance =
Euclidean(envelope_A , envelope_B)

Candidate pool becomes:

100 songs → ~20 songs
4️⃣ Layer 3 – Transition Continuity Optimization

This is the key innovation of your project.

We check whether:

Song A ending energy
≈
Song B starting energy
Transition Metric
transition_gap =
|mean(last_k(A)) - mean(first_k(B))|

Lower value means smoother transition.

Final Ranking Function

The next song score becomes:

score =
α * similarity
- β * shape_distance
- γ * transition_gap

Weights will be tuned experimentally.
